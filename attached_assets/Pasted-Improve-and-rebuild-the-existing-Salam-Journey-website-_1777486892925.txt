## 🎨 UI/UX Elevation Requirements

### General Design Rules:
- **Glassmorphism cards**: `background: rgba(255,255,255,0.7); backdrop-filter: blur(12px); border: 1px solid rgba(127,169,155,0.2);`
- **Organic shapes**: Use SVG blob/wave dividers between sections instead of straight lines
- **Micro-animations**: 
  - Cards lift on hover (`transform: translateY(-6px); box-shadow: 0 20px 40px rgba(90,138,128,0.15)`)
  - Staggered fade-in on scroll using IntersectionObserver
  - Smooth page-load entrance animations (stagger 100ms per element)
- **Decorative elements**: Subtle leaf SVG patterns, soft dot grids in section backgrounds
- **Buttons**: Pill-shaped (`border-radius: 50px`), with gentle pulse animation on primary CTAs
- **Section spacing**: Generous — min 100px vertical padding per section
- **Mobile-first**: Everything must look beautiful on 375px screens

### Navbar:
- Frosted glass effect: `backdrop-filter: blur(20px); background: rgba(250,245,232,0.85)`
- Logo: show the circular Salam Journey logo image
- Links with underline-slide hover animation
- Language toggle: pill toggle switch (AR / EN)
- Mobile: smooth hamburger menu that slides in from side

### 8. ⭐ BOOKING / SESSIONS PAGE — Full Calendar System

This is the most important section to build correctly.

**Layout**: 2-column — left/right: session info | calendar picker

**Session Info Panel:**
- Session types as selectable cards:
  1. جلسة فردية / Individual Session — 60 min
  2. حزمة ٣ جلسات / 3-Session Package
- When selected, card highlights with sage border and checkmark

**Interactive Calendar:**
- Build a full monthly calendar grid from scratch (no external library needed)
- Show current month with prev/next month navigation arrows
- Each day cell: clickable if available, greyed-out & unclickable if:
  - It's in the past
  - It's already booked (store booked dates in a JS array/localStorage)
- Available days: white with sage hover
- Selected day: filled sage green with white text
- Booked/unavailable days: `opacity: 0.35; cursor: not-allowed; text-decoration: line-through`
- Weekend days (Friday/Saturday for Arab users): slightly different shade

**Time Slots (appear after selecting a day):**
- Show time slots as pill buttons: `10:00 ص`, `12:00 م`, `2:00 م`, `4:00 م`, `6:00 م`
- Booked slots: greyed out, unclickable
- Selected slot: filled sage

**Booking Form (appears after selecting day + time):**
Animated slide-in form with fields:
- الاسم الكامل / Full Name
- البريد الإلكتروني / Email
- رقم الواتساب / WhatsApp Number
- موضوع الجلسة / Session Topic (select dropdown with options)
- ملاحظات إضافية / Additional Notes (optional textarea)
- Submit button: `تأكيد الحجز / Confirm Booking`

**On Submit:**
- Mark that date+time as booked in localStorage (so it persists on refresh)
- Show success message with animation: `تم الحجز بنجاح! سيتم التواصل معكِ خلال ٢٤ ساعة ✓`
- Booked slot now appears greyed-out immediately

## ⚙️ Technical Stack

- **React + Vite** (preferred) OR vanilla HTML/CSS/JS in one file
- No external UI libraries (build everything from scratch for full control)
- Google Fonts loaded via `<link>` in head
- All state managed with React useState/useEffect OR vanilla JS
- localStorage for: language preference + booked appointments
- IntersectionObserver for scroll animations
- Fully responsive: 375px (mobile) → 768px (tablet) → 1280px (desktop)

## 🚫 Strictly Avoid:
- Purple, blue, grey color schemes
- Flat/boring card designs with no depth
- Generic "AI website" look
- Hard-coded text without bilingual data attributes
- Any external calendar library (build it from scratch)
- Letting users type free text for booking times (must use calendar + time slot picker)
- Sharp corners anywhere (use border-radius generously)
